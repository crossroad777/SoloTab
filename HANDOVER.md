# SoloTab プロジェクト引継書 (更新日: 2026/03/21 最新版)

## 1. プロジェクトの現状と直近の目標

SoloTabは、アコースティックギターの音源やYouTube URLから高精度なタブ譜（MusicXML）を自動生成するシステムです。
現在の最大の目標は、**「Songsterrのようなクリーンで洗練されたUI/UXの実現」** と、**「MIDIのような無機質な音源から発生する、極端に細かく大量のゴーストノート（ゴミ音符）や不要なダイナミクス記号（f, ff等）をユーザー操作で駆逐し、クリーンな3連符アルペジオとして視覚化すること」** です。

---

## 2. 直近で実装・完了した機能（フロントエンド＆バックエンド）

### 2.1 UIの徹底的クリーンアップと「CUTスライダー」の実装 (App.jsx)

- Songsterr Plusに似せたものの、裏側が未実装だった大量のダミーボタン（シンセシス再生、ソロ、ミュート、カウントイン、メトロノーム、エディタ等）を**すべて完全削除**しました。
- 現在のプレイヤーバーは「再生/停止」「再生時間」「速度(SPEED)」「ループ」「CUTスライダー」「エクスポート(MusicXML)」「印刷」のみの非常にシンプルで機能する状態にシェイプアップされています。
- **CUTスライダー（ノイズ除去レベル調整）**: AIが過剰に拾いすぎた倍音やゴーストノートを、フロントエンドのシークバー操作で直感的に削ぎ落とす機能を実装しました（0%〜80%以上までスライダーで調整可能）。

### 2.2 バックエンドの強力なノイズゲートフィルターとクオンタイズ (tab_renderer.py)

- **ダイナミクス(f/ff)スパムの完全消去**: `f` や `ff` といった強弱記号が譜面に大量に印字されて真っ黒になる問題を解決するため、MusicXMLへのダイナミクス書き出しロジック（`<dynamics>`タグ）を行ごと完全に削除しました。
- **強制的3連符クオンタイズ (`_group_by_time`)**: 近接するノートを同一の拍（Chord）として強制的にグループ化し、タイムラインが16分音符などで細かく砕ける現象を防ぎ、綺麗なアルペジオとして整列させるロジックを強化しました。
- **動的トラック相対ノイズゲート**: 楽曲のノートが持つ「最小音量」と「最大音量」を動的に取得し、その幅（レンジ）に対して 0%~100% の割合でCUTスライダーの閾値を適用するフィルターを実装しました。これにより、どんな音源でも確実に「相対的な小音量ノイズのみ」を狙い撃ちして削除できるようになりました。

## 3. 直近で発生・修正した重大なバグ（引継ぎにおける超重要事項）

これらは全てコード上では**修正完了**していますが、サーバーの再起動が行われない限りユーザーの手元には反映されない性質のものでした。

1. **バックエンドのクラッシュ (500 Error)**
   - クオンタイズ処理中、`int(e["beat_pos"])` に対して `"4.0"` というfloat形式の文字列が渡され、`ValueError` で裏側がクラッシュする致命的なバグがありました。
   - **【修正済】**: `abs(float(e) - float(prev)) < 0.1` という安全なfloat比較に修正しました。現在、このエラーによるクラッシュは100%解消されています（ローカルで24万文字の正常なXML出力テスト完走済）。

2. **CUTスライダーのパーセンテージが効かない問題（数学的ミス）**
   - 以前の実装では、FretNet出力の最低音量が「55%（0.55）」であることを無視し、絶対値で「45%以下をカット」という処理にしていたため、実質的に1音もカットされない状態でした。
   - **【修正済】**: `min` `max` を取得しての割合計算（動的相対ゲート）に書き換えたため、現在はスライダーにリニアに反応してノートが激減します。

3. **Reactのクロージャ状態（古い値の送信）バグ**
   - CUTスライダーを離した瞬間(`onMouseUp`)に、Reactの非同期ステートの仕様により「一つ前の古いCUT数値」をバックエンドに毎回送ってしまうバグがありました。
   - **【修正済】**: `e.target.value` をイベントハンドラ内で直接 `parseFloat` してAPIに送るように修正しました。

4. **MusicXMLのブラウザキャッシュ問題**
   - 再計算が終わってフロントエンドが新しいMusicXMLを取得しに行っても、ブラウザがURLをキャッシュしており、古い `tab.musicxml` を使い回して同じ画面を描画していました。
   - **【修正済】**: `fetch` のURLクエリに `?t=${Date.now()}` を付与し、強制キャッシュバスターを導入しました（TabView.jsx）。

## 4. 残された課題とネクストアクション（ユーザー様・次のAIへ）

### ⚠️ Uvicornサーバーの再起動必須問題（最重要）

現在のアーキテクチャでは、`SoloTab一括起動.bat` で立ち上がる FastAPI (uvicorn) 側には `--reload` オプションがついておらず、**Pythonコードをいくら書き換えても、黒い画面（コマンドプロンプト）を一度手動で閉じて再起動しない限り、新しいノイズ除去機能が絶対に適用されません。**
これまでの「変わっていない」という現象の最大の原因は、【サーバーメモリ上にバグっていた古いPythonコードが残り続けたままになっていたこと】と、【上記の4つの実装バグが複合していたこと】です。

#### 【次に行うべきアクション】

次の作業を再開・検証する前に、**必ず** 以下の手順を踏んでください。

1. 現在開いている黒い画面（バックエンド・フロントエンドのターミナル）をすべて「×」ボタン等で閉じて完全に終了させる。
2. 再度 `SoloTab一括起動.bat` をダブルクリックし、クリーンな状態でサーバーを立ち上げ直す。
3. ブラウザ（`localhost:5174` 等）を F5 キーでハードリロードする。
4. CUTスライダーを操作し、ノイズが適切に消えるか（ノート数が減り、3連符が綺麗に表示されるか）を確認する。

コード側の準備とバグ修正は完全に終わっています。サーバーの再起動さえ行われれば、必ずクリーンな譜面が出力される状態にあります。
