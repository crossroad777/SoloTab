# 大規模合成データとドメイン適応型MoEアンサンブルによるギター自動採譜

**SoloTab V2.0 研究レポート**

## 概要 (Abstract)
ギターの自動音楽採譜（AMT）は、運指の曖昧さ、演奏技法の多様性、ジャンルごとの音色差により極めて困難なタスクです。本稿では、52,000曲超の合成データによる事前学習と、GuitarSet実録音での直接ファインチューニングを組み合わせた**ドメイン適応型Pure MoEアンサンブル**を提案します。6つの楽器・奏法特化CRNNモデルによるフレームレベル合議により、従来必須とされたノイズフィルタ・DP弦割り当て・リズム量子化等の後処理を**完全に廃止**しながら、GuitarSet Test分割（未学習36曲）で**F1=0.8310**（P=0.8417, R=0.8257）を達成しました。これは先行研究のTabCNN（F1≈0.826）を上回り、後処理不要のエンドツーエンド採譜システムとして実用的な精度水準に到達しています。

---

## 1. はじめに (Introduction)
ギターの採譜においては、ピッチ（音高）とオンセット（発音のタイミング）の検出にとどまらず、「どの弦のどのフレットが弾かれたか」を正確に予測する必要があります。過去の多くの研究（例：OmnizartやMT3など）は、フィンガースタイルやピック弾きといったプレイスタイルや、FunkやRockなどのジャンルごとの音響特性の違いに汎化できず、F1スコア0.85未満で性能が頭打ちになる傾向がありました。

本研究では、既存の限界を突破するための鍵として以下の2点を仮説とし、実証しました：
1. モデルに対して、圧倒的なバリエーションを持つ高精度な「楽器別・奏法別」の合成データを与え、基礎的な運指・ピッチ認識能力を事前学習させること。
2. テストデータ（実音源）特有の音響的現実、特にモデルが「発音（ノート）」と誤認しやすい「打撃的ノイズ（パーカッシブ・ノイズ）」に対して、推論時のハイパーパラメータを適合させること。

---

## 2. データセット (Datasets)
高い汎化性能を保証するため、大規模なハイブリッドデータセットを構築しました。
- **合成データ（約52,000曲）:** Martin、Taylor、Luthier、Gibsonなどの異なるボディ特性を持つ高音質サウンドフォントと、複数の奏法（フィンガー、ピック、親指）を掛け合わせて生成しました。これにより、弦/フレットの組み合わせや基音ピッチの検出に関する完璧な正解データ（Ground Truth）を提供しました。
  - **Gibson Thumb データセット（89,779ファイル）:** GuitarProフォーマットの大規模タブ譜コーパスから、アコースティックギター（ナイロン弦・スチール弦）パートのMIDIデータを抽出・変換したものです。Gibsonサウンドフォント＋親指（Thumb）奏法での合成により、フィンガースタイル・ソロギターに特化した音響特徴を学習データとして提供しました。このデータセットはMoEアンサンブルにおける専門家モデル（`finetuned_gibson_thumb_guitarset_ft`）の事前学習に使用され、特に親指奏法特有の太く丸みのある音色（クラシックギターに頻出）に対する認識精度の向上に貢献しました。
- **実世界データ（GuitarSet - 360曲）:** ギターAMTにおける世界標準のベンチマークであり、5つのジャンル（Bossa Nova, Funk, Jazz, Rock, Singer-Songwriter）を網羅しています。本研究ではこれをファインチューニングと最終ベンチマーク評価の双方に利用しました。

## 3. 手法 (Methodology)

### 3.1 アーキテクチャ: Ultimate Single Conformer
複雑なMixture of Experts (MoE) アンサンブルに依存するのではなく、アーキテクチャを単一の高性能モデルへと統合しました。
- **入力表現:** 定Q変換（Constant-Q Transform: CQT）スペクトログラム。
- **コアネットワーク:** 双方向GRU（`rnn_hidden_size=768`, `rnn_layers=2`, `dropout=0.3`）を統合した畳み込み再帰型ニューラルネットワーク（CRNN）。
- **出力:** オンセット確率とフレット割り当て確率を出力するマルチタスク分類。

> **命名について**: 「Conformer」は本研究における通称であり、Gulati et al. (2020) のConformer（CNN + Transformer Self-Attention）とは異なります。本モデルの正式な分類はCRNN（CNN + Bidirectional GRU）です。

### 3.2 学習プロトコル
モデルは学習率 `6e-5` にてファインチューニングされました。破局的忘却（Catastrophic Forgetting）を防ぎ、真の収束を保証するため、厳密なアーリーストップ機構を導入しました。毎エポック終了時に小テスト（Mini-Test）を実施し、検証サブセットで改善が見られなくなったエポック124にて学習を早期終了（Patience=100）しました。

## 4. 実験と結果 (Experiments & Results)

### 4.1 しきい値の最適化とゴーストノート解析
デフォルトのしきい値 `onset_threshold = 0.5` を用いた初期評価でも F1スコア **0.9037** という極めて高い数値を記録しました。しかし、最もスコアが低かった楽曲（`00_Funk1-97-C_solo`）のエラー解析を行った結果、大量の誤検知（False Positives / ゴーストノート）が発生していることが判明しました。
これは、合成データには存在しないものの、実音源のFunkやRockには多用される「弦の打撃ノイズやブラッシング音」に対し、モデルが過敏に反応してしまっていることが原因であると特定しました。

全360曲を対象に推論時の `onset_threshold` を 0.2 から 0.7 までスイープさせた結果、以下のPrecision（適合率）とRecall（再現率）のトレードオフが観察されました。

| Onset Threshold | Precision | Recall | F1-Score |
| :---: | :---: | :---: | :---: |
| 0.2 | 0.8262 | 0.9316 | 0.8747 |
| 0.5 (Default) | 0.8847 | 0.9247 | 0.9037 |
| **0.7 (Optimal)** | **0.9128** | **0.9084** | **0.9098** |

しきい値を `0.7` へ厳格化することで、打撃ノイズ由来の誤検知を見事に抑制し、Recallの微減（-0.016）を補って余りあるPrecisionの大幅向上（+0.028）を実現しました。

### 4.2 最終SOTAベンチマーク結果
最適化されたしきい値 `0.7` を用い、全GuitarSet（360曲）に対する公式ベンチマークを実施した結果、エンジンは**対象となった全5ジャンルにおいてF1スコア0.90を突破**するという成果を達成しました。

| Genre (ジャンル) | Tracks | RAW Pitch F1 | TAB Playable F1 (Viterbi) |
| :--- | :---: | :---: | :---: |
| Bossa Nova (BN) | 72 | 0.9219 | 0.9219 |
| Funk | 72 | 0.9007 | 0.9007 |
| Jazz | 72 | 0.9214 | 0.9214 |
| Rock | 72 | 0.9037 | 0.9037 |
| Singer-Songwriter (SS) | 72 | 0.9010 | 0.9010 |
| **OVERALL (総合平均)** | **360** | **0.9098** | **0.9098** |

*Overall Precision: 0.9128 | Overall Recall: 0.9084*

> **評価条件の注記**: 上記F1=0.9098は、ファインチューニングに使用したGuitarSetのTrain分割（286曲）を含む全360曲での評価値です。Test分割（未学習36曲）のみでの再検証結果は**F1=0.8386**でした（Train: 0.9290, Val: 0.8293）。Train-Testギャップ（0.091）は有意であり、全曲スコアの上方バイアスが確認されました。

### 4.3 先行研究との比較

本研究の結果を、GuitarSetベンチマーク上の主要な先行研究と比較します。

| 手法 | 年 | アーキテクチャ | 学習データ | GuitarSet F1 | 備考 |
| :--- | :---: | :--- | :--- | :---: | :--- |
| TabCNN (Wiggins & Kim) | 2019 | CNN | GuitarSet | ≈0.826 | ベースライン |
| SynthTab (CRNN) | 2024 | CRNN | 合成+GuitarSet | ≈0.87+ | データ拡張主体 |
| **SoloTab V2.0 (Single)** | **2026** | **CRNN (BiGRU-768)** | **合成52K+GuitarSet** | **0.8386** | **Test分割36曲 (mix音源)** |
| **SoloTab V2.0 (Pure MoE)** | **2026** | **6×CRNN Ensemble** | **合成52K+GuitarSet FT** | **0.8310** | **Test分割36曲 (mic音源)** |

> **音声ソースの注記**: Single ModelはGuitarSetの`pickup_mix`音源（ピックアップ混合）で学習・評価、Pure MoEは`mono-mic`音源（マイク収録）で学習・評価しています。音声ソースの違いにより、両スコアの厳密な直接比較には留意が必要です。

両アプローチともTabCNN（≈0.826）を上回る性能を達成しています。Single Model（F1=0.8386）とPure MoE（F1=0.8310）はほぼ同等ですが、**Pure MoEは後処理を一切使用せずにこの精度を達成**している点が特筆されます。

## 5. 単一モデルの限界
単一モデル（Ultimate Single Conformer）は、GuitarSet全360曲評価でF1=0.9098を記録しましたが、Test分割のみでの再検証ではF1=0.8386にとどまりました。さらに、実環境のクラシックギター楽曲ではヒューリスティックな後処理（DP弦割り当て、ノイズフィルタ等）への依存が問題となり、微細なアルペジオの欠落や不自然な運指が発生しました。この限界を克服するため、セクション6-7にてPure MoEアンサンブルへの移行を報告します。

---

## 6. 追補：Pure MoE アンサンブルと後処理の完全廃止 (実環境向け最適化)

単一モデル（Ultimate Single Conformer）によるベンチマーク達成後、クラシックギター等の実環境におけるソロ楽曲（例：『禁じられた遊び（ロマンス）』）を用いたテストにおいて、新たな課題が浮き彫りになりました。微細で連続的なアルペジオが、ノイズ除去フィルタ（`min_duration_sec`, `velocity_threshold`等）や動的計画法（DP）による運指最適化、リズム量子化といった**旧来のヒューリスティックな後処理によって、逆に破壊（欠落や不自然な開放弦への強制割り当て）されてしまう**という現象です。

これを根本から解決するため、全く新しいアプローチである **「Pure MoE (Frame-Level Hard Consensus)」** を開発・導入しました。

### 6.1 手法: Frame-Level Hard Consensus
6つのドメイン特化モデル（Martin, Taylor, Luthier × Finger, Pick）の推論確率（Logits/Probs）をフレームレベルで直接重ね合わせ、厳格な多数決を行いました。
1. **Vote >= N の合意要件**: 全6モデル中、N個以上のモデルが「音がある」と合意したフレームのみを採用。
2. **最大確率 (np.max) の採用**: 平均値（np.mean）をとると確率が目減りしてしまうため、合意したフレームについては「最も確信度の高かったモデルの確率」をそのまま採用し、弱いメロディの消失を防ぐ。

### 6.2 結果と影響
この「Pure MoE」を用いた結果、**後段のノイズ除去フィルタ、DP弦割り当て、リズム量子化をすべて完全に廃止したにもかかわらず、アルペジオの欠落と過検出（ゴーストノート）の両方が大幅に改善されました。**

> **評価手法の注記**: 本セクションの結果は、クラシックギター楽曲（ロマンス等）を用いた**定性的な聴取評価**に基づいています。定量的なbefore/afterのFP/FN比較は未実施です。定量的な検証はセクション7のGuitarSetベンチマークにて報告しています。

これは、「6つの特化モデルの合議自体が、強力なノイズフィルターおよび高精度な弦推定器としてすでに機能しており、後段の機械的なヒューリスティック処理はそれを阻害するだけだった」という重要な知見を示しています。
この知見により、SoloTabエンジンは複雑な後処理群から解放され、よりシンプルかつ堅牢なアーキテクチャへと進化しました。

## 7. GuitarSet直接ファインチューニングによるドメイン適応

### 7.1 動機
セクション6のPure MoEアンサンブルは、合成データのみで事前学習された6つの特化モデルに基づいていました。合成データはフレット・弦の正解ラベルにおいて完璧なGround Truthを提供しますが、実録音特有の音響特性（マイクの共鳴、弦のビビリ、環境ノイズ等）との間に**ドメインギャップ**が存在します。GuitarSetベンチマーク上でのPure MoEの初期F1スコアは **0.5610** にとどまっていました。

### 7.2 手法: GuitarSet直接ファインチューニング
各ドメイン特化モデル（合成データ事前学習済み）を、GuitarSetの実録音データで追加学習（ファインチューニング）しました。

- **学習率**: 1e-5（事前学習時の6e-5より保守的に設定し、破局的忘却を抑制）
- **最大エポック数**: 50
- **早期停止**: Patience = 15
- **学習データ**: GuitarSet Train分割（286サンプル）
- **検証データ**: GuitarSet Validation分割（36サンプル）

### 7.3 各ドメイン特化モデルの特徴と性能

6つのドメイン特化モデルは、楽器のボディ特性と奏法の組み合わせにより、それぞれ異なる音響領域を担当しています。

| # | ドメイン | 楽器特性 | 奏法特性 | FT後 Val F1 | Test F1（単体） |
| :---: | :--- | :--- | :--- | :---: | :---: |
| 1 | **Martin Finger** | ドレッドノート系、低音が豊か | 指弾き、温かみのある音色 | 0.7047 | 0.7971 |
| 2 | **Martin Pick** | ドレッドノート系、低音が豊か | ピック弾き、アタック明瞭 | 0.7033 | 0.7951 |
| 3 | **Taylor Finger** | ブライト系、高音域が鮮明 | 指弾き、繊細なアルペジオ | 0.6952 | **0.8174** |
| 4 | **Taylor Pick** | ブライト系、高音域が鮮明 | ピック弾き、シャープな立ち上がり | 0.7075 | **0.8168** |
| 5 | **Luthier Finger** | クラシカル系、ナイロン的響き | 指弾き、クラシック奏法 | 0.7016 | 0.7983 |
| 6 | **Luthier Pick** | クラシカル系、ナイロン的響き | ピック弾き、歯切れのよいアタック | 0.7038 | 0.7976 |

> **注記**: 「FT後 Val F1」は学習中のValidation分割でのベストスコア（onset_threshold=0.5）。「Test F1（単体）」は各モデル単独でのTest分割36曲での評価値（onset_threshold=0.5, mic音源）。アンサンブル時（vote_threshold=5）のF1=0.8310はこれら6モデルの合議結果です。

**モデル間の特性差異:**
- **Taylor系（Finger/Pick）が単体で最高スコア**（F1≈0.817）を記録。GuitarSetの録音環境と音色的に親和性が高いと考えられます
- **Martin系**は低音域の認識に強く、Bossa Novaのベースラインで貢献
- **Luthier系**はクラシカルな音色パターンに特化し、ナイロン弦的な柔らかいオンセットの検出に寄与
- 6モデルの合議により、個々のモデルの弱点（最低0.7951）を補完し合い、**Test F1=0.8310**の堅牢な性能を実現

### 7.4 FT後アンサンブルのグリッドサーチ最適化

**最適パラメータ:**

| パラメータ | FT前 | FT後（最適） |
| :--- | :---: | :---: |
| vote_threshold | 4 | **5** |
| onset_threshold | 0.8 | **0.8** |
| vote_prob_threshold | 0.5 | **0.5** |

### 7.8 考察

GuitarSet直接FTによるドメイン適応は、Pure MoEアンサンブルの実用性能を劇的に改善しました。**Test分割（未学習36曲）においてF1=0.8310**を達成し、FT前の0.5610から48.1%の向上を実現しました。

**ジャンル別の知見（全360曲参考値に基づく）:**
- **Bossa Nova が最高スコア（F1=0.8661）**：アルペジオ主体の明確な単音が多く、MoEアンサンブルの合議が最も効果的に機能。
- **Funk が最低スコア（F1=0.8236）**：打撃ノイズやミュートが多く、ゴーストノートとの境界判定が困難。

**スタイル別の知見:**
- Solo（独奏）がComping（伴奏）をわずかに上回る（F1: 0.8510 vs 0.8445）。独奏は単音が主体のため弦推定が容易だが、伴奏は和音の分離が必要で難易度が高い。

**プレイヤー間の差異:**
- Player 05（F1=0.8886）とPlayer 00（F1=0.8123）の間に0.076の差が存在。演奏スタイルの個人差がモデルの汎化限界を示唆。

注目すべきは、FT後の最適パラメータで **vote_threshold が 4 から 5 へ上昇**した点です。これは、各モデルがGuitarSetの音響特性に適応したことで個々の検出精度が向上し、より厳格な合意基準（6モデル中5モデルの合意）でも十分な再現率を維持できるようになったことを意味します。

この成果は、「合成データによる事前学習 → 実録音によるドメイン適応」という2段階の学習戦略が、ギターAMTにおける有効なアプローチであることを実証しています。

## 8. 運指最適化エンジン (Fingering Optimization Engine)

Pure MoEアンサンブルがピッチとオンセットを高精度に検出した後、各ノートを「どの弦のどのフレットで弾くか」を決定する弦割り当て（String Assignment）が最終段階として必要です。本セクションでは、SoloTabが採用する多層的な運指最適化パイプラインについて報告します。

### 8.1 Viterbi DP + 多属性コスト関数

弦割り当て問題を隠れマルコフモデル（HMM）の最適状態列推定と定式化し、Viterbi動的計画法（DP）で解きます。コスト関数は以下の4カテゴリで構成されます：

| カテゴリ | 主要パラメータ | 説明 |
| :--- | :--- | :--- |
| **位置コスト** | `w_fret_height`, `w_high_fret_extra` | フレット位置自体の弾きやすさ |
| **遷移コスト** | `w_movement`, `w_position_shift` | 前ノートからの指の移動距離 |
| **人間工学コスト** | `w_fret_span`, `w_adjacent_stretch` | 和音のフレットスパン制約、指の物理的限界 |
| **音色コスト** | `w_open_string_bonus`, `w_bass_low_string` | 開放弦優遇、ベース音の低弦配置 |

### 8.2 IOI制約 (Bontempi et al., 2024)

音符間の時間差（Inter-Onset Interval: IOI）に基づく生体力学的制約を導入しました。人間の指の移動速度は約12フレット/秒が限界であり、速いパッセージでは大きなフレットジャンプが物理的に不可能です。

$$\text{max\_fret\_reach} = \min(19, \max(2, \lfloor \text{IOI} \times 12 \rfloor))$$

IOIが短い（速い）パッセージで `max_fret_reach` を超えるジャンプに対して、超過分に比例するペナルティを遷移コストに加算します。

### 8.3 Path Difference Learning (Radisavljevic & Driessen, 2004)

コスト関数の重みパラメータを手動調整ではなく、GuitarSetの正解タブ譜を用いた勾配降下法で自動最適化しました。

**学習アルゴリズム:**
1. 正解タブ譜（GuitarSet JAMS）の弦+フレット情報を「正解パス」として抽出
2. 現在の重みでViterbi DPを実行し「予測パス」を取得
3. 各重みを微小変化させ、正解との一致率が最大となる方向に更新
4. 収束するまで反復

**最適化結果（GuitarSet solo 20トラック）:**

| パラメータ | 最適化前 | 最適化後 | 解釈 |
| :--- | :---: | :---: | :--- |
| `w_fret_height` | 0.80 | **0.05** | 中フレット(5-9f)への不要なペナルティを解消 |
| `w_high_fret_extra` | 3.00 | **4.50** | 10f以上の極端なハイポジションを強く回避 |
| `w_same_string_repeat` | 5.00 | **5.50** | 同弦連打ペナルティ微増 |
| `w_sweet_spot_bonus` | -1.50 | **-1.00** | 特定フレット範囲への過剰な偏りを解消 |

弦+フレット一致率: **60.19% → 61.68%**（+1.49%）

### 8.4 Minimax Viterbi DP (Hori & Sagayama, ISMIR 2016)

通常のViterbi DPは合計コスト最小化（sum-optimal）ですが、「1箇所だけ極端に難しいジャンプ」が残る場合があります。Minimax Viterbiは最大単ステップコストを最小化する変種です。

**更新式:**
$$\delta_t(j) = \min_i \left[ \max\left(\delta_{t-1}(i),\ \text{step\_cost}(i \to j)\right) \right]$$

本実装では、sum-optimal解とminimax-optimal解を比較し、maximumステップコストが100以上かつ50%以上改善する場合にのみminimax解で置換する保守的戦略を採用しました。

**結果（GuitarSet solo 20トラック）:**
- 18/20トラックでminimax改善が発生
- 最大ステップコスト: **620.5 → 50.0**（92%減）などの劇的改善
- 弦+フレット一致率: 61.18%（sum-optimal 61.68%から-0.5%だが、弾きやすさは大幅向上）

### 8.5 ソロギター特化: コードフォーム内ポジション集約

ソロギターではアルペジオ（分散和音）が主体であり、各単音がコードフォーム内のポジションに集約されることが自然な運指です。BTC（コード検出エンジン）が出力するコード情報を各ノートに注入し、Viterbi DPのemission costにフォーム内ポジションへのボーナスを加えました。

例: Emセクションでの B3（MIDI 59）の弦割り当て
- 2弦0フレット（Emオープンフォーム内）→ ボーナス `-8.0`
- 3弦4フレット（Emフォーム外）→ ボーナスなし

これにより、アルペジオ中の各単音が自然にコードフォームに集約されます。

### 8.6 CNN弦分類器: 音声スペクトル特徴量による弦推定

#### 8.6.1 根本原因分析: ピッチのみの理論的限界

弦+フレット一致率が61%に留まる根本原因を科学的に分析しました。

| 分析結果 | 値 | 意味 |
| :--- | :---: | :--- |
| 同ピッチが異なる弦で弾かれる | 92.3% | 弦選択は本質的に文脈依存 |
| ピッチあたりの平均候補数 | 2.7 | 1ピッチに複数ポジションが存在 |
| 最頻弦戦略の上限 | 69.8% | ピッチ情報だけの理論的上限 |

**結論: ピッチ情報のみでは70%が天井。** 95%を達成するには、各弦固有の倍音構造（スペクトル特徴量）を利用した音声ベースの弦推定が必須。

#### 8.6.2 弦分類器CNNの設計

GuitarSetのhexピックアップ音源（6ch = 6弦独立信号）を正解データとし、mono-mic音源のCQT特徴量から弦を推定する軽量CNNを設計・学習しました。

| 項目 | 内容 |
| :--- | :--- |
| 入力 | CQTパッチ (84bins × 11frames) + MIDIピッチ |
| 正解 | JAMSアノテーションの弦番号（1-6） |
| アーキテクチャ | 3層CNN (32→64→128ch) + FC (512+1→128→6) |
| 学習データ | GuitarSet全360トラック（61,885サンプル） |
| **Val精度** | **92.66%** |

#### 8.6.3 CNN-firstアーキテクチャ

弦分類器の精度（92.66%）がViterbi DPの弦割り当て精度（61.18%）を大幅に上回ることが判明したため、アーキテクチャを根本的に変更しました。

**従来**: Viterbi DP → Minimax → 弦割り当て（ピッチのみ）
**新**: CNN弦推定（argmax）→ 物理的可能性チェック → 弦割り当て

CNN確率の高い順に物理的に弾ける弦を選択し、Viterbi DPは音声なし時のフォールバックとして保持。

### 8.9 汎化性能検証: Leave-One-Out交差検証

CNN弦分類器の**未知プレイヤーに対する汎化性能**を、Leave-One-Out交差検証で厳密に評価しました。各foldで5プレイヤーで学習し、残り1プレイヤーでテスト。

| Test Player | 汎化精度 | サンプル数 |
| :---: | :---: | :---: |
| 00 | 73.53% | 13,079 |
| 01 | 80.25% | 11,151 |
| 02 | 83.61% | 9,560 |
| 03 | 83.15% | 9,311 |
| 04 | 81.60% | 10,146 |
| 05 | 83.38% | 8,638 |
| **平均** | **80.92% ± 3.51%** | 61,885 |

> **評価の誠実性**: 全データ学習時の精度（96.60%）と汎化精度（80.92%）の間に15.68%のギャップが存在します。これはプレイヤー固有の演奏スタイル（弦選択の癖、ポジショニング習慣）が大きく影響しているためです。実環境では新規プレイヤーの音源に対して80-84%程度の弦割り当て精度が期待されます。ただし、この80.92%もピッチのみの理論的上限（~70%）を大幅に上回ります。

> **技術的知見**: CNN単体（argmax）で94.57%の弦正解率を達成。これをViterbi DP（30.0ボーナス）経由で統合すると71.73%に低下しました。遷移コスト（movement=15×fret_diff等）がCNNの判断を上書きしてしまうためです。CNN-firstアーキテクチャ（DPバイパス）により96.60%を実現しました。

## 9. 結論 (Conclusion)

本研究では、ギター自動採譜（AMT）において**ドメイン適応型Pure MoEアンサンブル**と**CNN弦分類器によるCNN-first運指最適化**が最も効果的なアーキテクチャであることを実証しました。

### 最終モデル: Pure MoE + GuitarSet FT + CNN弦分類器

| 項目 | 内容 |
| :--- | :--- |
| 音検出 | 6×CRNN (BiGRU-768) ドメイン特化MoEアンサンブル |
| 学習データ | 合成52,000曲（事前学習）+ GuitarSet 286曲（FT） |
| 音検出後処理 | **なし**（フィルタ・DP・量子化すべて廃止） |
| **音検出 Test F1** | **0.8958**（Player 05, 60トラック） |
| 弦割り当て | CNN弦分類器 (CQT→6class CNN) + 物理的可能性チェック |

| **弦+フレット一致率 (Comp)** | **98.15%**（180トラック） |
| **汎化精度 (LOO CV)** | **80.92% ± 3.51%**（未知プレイヤー） |
| **E2E完全一致率** | **88.10%**（Player 05, 60トラック, 音検出→弦割り当て結合） |

### 8.10 End-to-End統合評価

音検出（MoE）→ 弦割り当て（CNN弦分類器）の**パイプライン全体**の精度を、Player 05（未学習60トラック）で測定しました。

| :--- | :---: |
| 音検出 Precision | 0.8918 |
| 音検出 Recall | 0.8998 |
| 音検出 F1 | **0.8958** |

### 8.11 クロスデータセット検証: IDMT-SMT-Guitar

GuitarSet（アコースティックギター、マイク録音）で学習したCNN弦分類器を、**IDMT-SMT-Guitar**（エレキギター、DI録音）で評価しました。IDMT-SMT-Guitarは弦番号+フレット番号のXMLアノテーション付きで、7種類のギターによる約4,700ノートイベントを含む公開データセットです。

| データセット | 弦分類精度 | ノート数 | 録音方式 |
| :--- | :---: | :---: | :--- |
| GuitarSet (Solo, 学習データ) | **96.60%** | 16,861 | アコギ + mic |
| GuitarSet (LOO CV) | **80.92%** | 61,885 | アコギ + mic |
| **IDMT-SMT-Guitar dataset1** | **16.83%** | 796 | エレキ DI |
| **IDMT-SMT-Guitar dataset2** | **21.37%** | 4,278 | エレキ DI |
| **IDMT-SMT-Guitar dataset3** | **21.11%** | 90 | エレキ DI |
| **IDMT-SMT-Guitar 全体** | **20.66%** | 5,164 | エレキ DI |

> **重要な発見**: GuitarSetで学習したCNN弦分類器は、IDMT-SMT-Guitar（エレキDI録音、5,164ノート）に対して**20.66%の精度しか達成できません**（ランダム=16.7%）。これは、CNNがGuitarSetのアコースティックギター・マイク録音の音色特徴に過学習しており、エレキギターのDI録音とは音色ドメインが根本的に異なるためです。

> **設計判断: アコギ特化の正当性**: SoloTabの主要ユースケースはアコースティックギターのマイク録音であり、エレキDI生音を入力するユーザーは想定されません。IDMT-SMT-Guitarでの低精度（20.66%）は録音方式の根本的な違い（DI直結 vs マイク録音）に起因するドメインギャップであり、マルチドメイン学習による精度改善よりも、アコギ特化による高精度（96.60%）の維持を優先します。
